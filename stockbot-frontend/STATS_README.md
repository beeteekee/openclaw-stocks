# 股票查询统计功能说明

## 功能概述

新增了每天查询股票的数量和查询次数的统计功能，并在网页底部以横向滚动的方式展示前一天的统计数据。

## 功能特性

### 1. 自动统计
- 每次用户查询股票时，系统会自动记录统计信息
- 统计内容包括：
  - 查询日期
  - 查询次数（每次点击"分析"按钮计数一次）
  - 股票数量（当天查询过的不同股票代码数量）

### 2. 数据存储
- 统计数据存储在 `stats.json` 文件中
- 数据格式：
```json
{
  "daily": {
    "2026-02-15": {
      "date": "2026-02-15",
      "queryCount": 10,
      "stockCount": 5,
      "stocks": {
        "002594.SZ": true,
        "600519.SH": true
      }
    }
  }
}
```

### 3. 前端展示
- 在网页底部添加了一个横向滚动条
- 展示**前一天**的统计数据
- 滚动条内容：
  - 日期：前一天的日期
  - 查询次数：前一天的总查询次数
  - 股票数量：前一天查询的不同股票数量
- 鼠标悬停时暂停滚动

### 4. API 端点

#### 记录查询
```
POST /api/stats/record
Content-Type: application/json

{
  "stockCode": "002594.SZ"
}
```

#### 获取前一天的统计数据
```
GET /api/stats/previous

响应：
{
  "date": "2026-02-15",
  "queryCount": 10,
  "stockCount": 5
}
```

## 数据说明

### 查询次数
每次用户点击"分析"按钮或按回车键查询股票时，计数器+1
- 同一只股票查询多次，查询次数累加
- 股票数量不重复计算

### 股票数量
统计当天查询过的**不同**股票代码数量
- 相同股票代码多次查询，只计为1个
- 统计基于规范化后的股票代码（包含.SZ或.SH后缀）

### 前一天数据
- 如果前一天的统计数据不存在，显示：
  - 日期：昨天的日期
  - 查询次数：0
  - 股票数量：0
- 每天零点后，会自动切换显示前一天的数据

## 技术细节

### 后端实现
- 使用Node.js + Express
- 统计数据存储在JSON文件中（stats.json）
- 提供RESTful API进行统计记录和查询

### 前端实现
- 纯CSS动画实现横向滚动效果
- 使用fetch API调用后端统计接口
- 页面加载时自动获取前一天统计数据

### 数据持久化
- 统计数据保存在文件系统
- 服务器重启后数据不会丢失
- 支持跨天统计和历史数据查询

## 注意事项

1. **数据文件权限**
   - 确保`stats.json`文件有读写权限
   - 文件会自动创建在`stockbot-frontend`目录

2. **时间同步**
   - 统计基于服务器系统时间
   - 确保服务器时间准确

3. **并发处理**
   - 文件读写操作可能有并发问题
   - 建议单线程使用，如需高并发可改用数据库

4. **数据备份**
   - 定期备份`stats.json`文件
   - 重要历史数据建议导出保存

## 故障排查

### 统计不更新
- 检查`stats.json`文件权限
- 查看服务器日志
- 确认API调用成功

### 显示"加载中..."
- 检查网络连接
- 确认服务器正常运行
- 查看浏览器控制台错误

### 滚动不显示
- 确认CSS样式正确加载
- 检查浏览器兼容性
- 查看元素是否被隐藏
