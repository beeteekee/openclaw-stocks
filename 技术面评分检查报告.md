# 技术面评分检查报告

## 检查目的
对照养家心法V8.0的技术面评分标准，检查当前实现是否严格按照标准计算。

## ❌ 发现的问题

### 问题1：250日均线评分严重不符合标准

#### 养家心法标准（SOUL.md 第2.1节）

| 条件 | 得分 | 说明 |
|------|------|------|
| 股价 > 250日均线 且 股价/250日均线 > 1.05 | 100分 | 长线强势 |
| 股价 > 250日均线 且 1.02 < 股价/250日均线 ≤ 1.05 | 80分 | 长线强势 |
| 股价 > 250日均线 且 股价/250日均线 ≤ 1.02 | 60分 | 刚站上长线 |
| 股价 < 250日均线 且 股价/250日均线 > 0.98 | 40分 | 刚跌破长线 |
| 股价 < 250日均线 且 0.95 < 股价/250日均线 ≤ 0.98 | 20分 | 长线弱势 |
| 股价 < 250日均线 且 股价/250日均线 ≤ 0.95 | **0分** | 长线弱势 |

#### 当前代码实现（technical_analysis.py）

```python
if close_price > ma250:
    # 股价在MA250上方
    if ratio > 1.05:
        ma250_score = 100
    elif ratio > 1.02:
        ma250_score = 80
    else:
        ma250_score = 60
else:
    # 股价在MA250下方（长期下跌）
    # 根据距离MA250的远近给分，不是直接0分
    if ratio > 0.9:
        ma250_score = 40  # 接近MA250，有反弹可能
    elif ratio > 0.8:
        ma250_score = 30  # 中等下跌
    elif ratio > 0.6:
        ma250_score = 10  # 深度下跌
    else:
        ma250_score = 5   # 极度下跌，但仍给基础分
```

#### 问题分析

1. **阈值不匹配**：
   - 标准：0.98、0.95
   - 代码：0.9、0.8、0.6
   - **结论：完全不符合标准**

2. **跌破MA250后给分过高**：
   - 标准：ratio ≤ 0.95 应该得 **0分**
   - 代码：ratio > 0.6 给10分，ratio ≤ 0.6 还给5分
   - **结论：长期弱势股票仍然得分，违背"低于250日均线=长线弱势"原则**

3. **缺少0分条件**：
   - 标准明确要求跌破0.95直接0分
   - 代码最低只给5分
   - **结论：没有真正的0分，违背养家心法原则**

#### 影响

- **误判风险**：长期弱势（跌破MA250）的股票仍然可能得到高分
- **选股风险**：可能选中长线弱势的股票，违背"趋势为王原则"
- **偏离原则**：不符合养家心法第2条"股价必须高于250日均线，低于250日均线=长线弱势，必须回避"

### 问题2：缠论买点判断逻辑不完整

#### 养家心法标准（SOUL.md 第2.3节）

**一类买点：**
- 下跌趋势中，出现背驰信号
- 股价创新低，但MACD等指标未创新低（底背驰）
- **当天就是底分型确认日**

**二类买点：**
- 一类买点后的回抽，不破前低
- 形成次级别走势类型
- **当天MACD金叉（DIF上穿DEA）**

**三类买点：**
- 突破前高后的回踩确认
- 站在中枢上方
- **当天就是突破确认日**

#### 当前代码实现（technical_analysis.py）

**一类买点：**
```python
if is_divergence and is_new_low and is_macd_not_new_low and is_bottom_fenxing:
    return {
        'type': 1,
        'score': 100,
        'desc': '一类买点：下跌趋势中出现底背驰...'
    }
```

**二类买点：**
```python
if is_golden_cross and is_not_break_prev_low and is_above_prev_bottom:
    return {
        'type': 2,
        'score': 80,
        'desc': '二类买点：MACD金叉确认...'
    }
```

**三类买点：**
```python
if is_break_prev_high and is_above_zhongshu and is_break_today:
    return {
        'type': 3,
        'score': 60,
        'desc': '三类买点：突破前高后回踩确认...'
    }
```

#### 问题分析

1. **底分型判断过于宽松**：
   - 当前：只检查`latest['bottom_fenxing'] == 1`
   - 问题：只要最后一根K线是底分型就算，没有检查是否是"确认日"
   - **结论：可能把前几天已经形成的底分型当成当天的买点**

2. **缺少"当天确认"的严格判断**：
   - 标准要求："当天就是底分型确认日"、"当天MACD金叉"、"当天就是突破确认日"
   - 代码只检查了条件满足，没有严格确认"当天"
   - **结论：可能把历史买点当成当天的买点**

3. **回抽判断可能不准确**：
   - 当前：`is_not_break_prev_low = latest['low'] > prev_low`
   - 问题：只检查最近5天，可能不够
   - **结论：判断回抽不破前低的逻辑可能不完整**

#### 影响

- **误判买点**：可能把已经过去的买点当成当天的买点
- **错过买点**：当天的买点可能因为判断逻辑不完整而错过
- **偏离原则**：不符合养家心法"当天数据满足买点条件才算真正的买点"

### 问题3：技术面综合得分计算不符合标准

#### 养家心法标准（SOUL.md 第2.2节）

```
技术面综合得分 = 250天均线得分 × 50% + 缠论买点得分 × 50%
```

#### 当前代码实现（technical_analysis.py）

```python
# === 技术面综合得分 ===
# 技术得分满分100分：250日均线50分 + 缠论买点50分
tech_score = ma250_score + chanlun_result['score']
```

#### 问题分析

1. **权重应用错误**：
   - 标准：250日均线得分 × 50%
   - 代码：直接相加 `ma250_score + chanlun_result['score']`
   - **结论：代码注释说"250日均线50分 + 缠论买点50分"，但实际实现没有应用50%权重**

2. **示例对比**：
   - 250日均线得分：80分
   - 缠论买点得分：80分（二类买点）

   **标准计算：**
   ```
   技术面综合得分 = 80 × 50% + 80 × 50% = 40 + 40 = 80分
   ```

   **代码计算：**
   ```python
   tech_score = 80 + 80 = 160分
   ```

3. **得分上限问题**：
   - 标准：技术面综合得分应该 ≤ 100分
   - 代码：可能达到200分（100+100）
   - **结论：得分可能超出标准范围**

#### 影响

- **得分虚高**：技术面得分可能远超100分
- **误导决策**：投资者可能误以为技术面很强，实际得分计算有误
- **偏离标准**：完全不按照养家心法的计算公式

### 问题4：stock_service.py中多次重复计算

#### 代码问题

```python
# 第1229行：使用technical_analysis.py返回的tech_score
tech_score = tech_result['tech_score']

# 第1266行：重新计算short_term_score
short_term_score = (tech_score * 0.8) - market_cap_penalty

# 第1285行：又重新计算
if current_emotion == 'unknown':
    short_term_score = tech_score * 0.8  # 仅使用技术面评分
```

#### 问题分析

1. **tech_score的含义不清晰**：
   - 是technical_analysis.py计算的100分制得分吗？
   - 还是250日均线得分（0-100）+ 缠论买点得分（0-100）？

2. **重复计算逻辑**：
   - 第1266行：`(tech_score * 0.8) - market_cap_penalty`
   - 第1285行：`tech_score * 0.8`
   - **结论：技术面得分在短期评分中只占80%，但技术面综合得分本身已经可能有错误**

#### 影响

- **计算混乱**：技术面得分的计算逻辑不清晰
- **难以维护**：多处重复计算，容易出错

## ✅ 符合标准的部分

1. ✅ 250日均线在MA250上方的评分逻辑（1.05、1.02、1.02以下）
2. ✅ 缠论买点的基础判断逻辑（背驰、金叉、突破）
3. ✅ 缠论买点评分标准（一类100分、二类80分、三类60分）

## 修复方案

### 修复1：250日均线评分

```python
if len(df) >= 250:
    ma250 = df['close'].tail(250).mean()
    ratio = close_price / ma250

    if close_price > ma250:
        # 股价在MA250上方
        if ratio > 1.05:
            ma250_score = 100
        elif ratio > 1.02:
            ma250_score = 80
        else:
            ma250_score = 60
    else:
        # 股价在MA250下方（严格按照标准）
        if ratio > 0.98:
            ma250_score = 40  # 刚跌破长线
        elif ratio > 0.95:
            ma250_score = 20  # 长线弱势
        else:
            ma250_score = 0   # 长线弱势，必须回避
else:
    ma250 = None
    ma250_score = 0
```

### 修复2：缠论买点判断

**增强"当天确认"的判断逻辑**：
- 底分型：不仅检查`bottom_fenxing == 1`，还要检查是否是"确认日"
- 金叉：不仅检查`DIF > DEA`，还要检查是否是"当天金叉"
- 突破：不仅检查`close > prev_high`，还要检查是否是"当天突破"

### 修复3：技术面综合得分

```python
# 技术面综合得分 = 250日均线得分 × 50% + 缠论买点得分 × 50%
tech_score = (ma250_score * 0.5) + (chanlun_result['score'] * 0.5)
```

### 修复4：stock_service.py使用正确的得分

```python
# 使用修复后的技术面综合得分（0-100分）
tech_score = tech_result['tech_score']
```

## 结论

**当前技术面评分计算存在严重问题，必须按照上述方案进行修复。**

核心问题：
1. ❌ 250日均线评分完全不符合标准（阈值错误、跌破仍然给分）
2. ❌ 缠论买点判断不完整（缺少"当天确认"的严格判断）
3. ❌ 技术面综合得分计算错误（没有应用50%权重）

严格按照养家心法修复这些问题后，技术面评分才能准确反映股票的技术状态。
