# 养家心法选股逻辑 - 评分系统检查报告

## 检查目的
对照养家心法V8.0的12条核心原则，检查当前评分系统的合理性和科学性。

---

## 核心问题

### ❌ 问题1：财务评分逻辑严重不符合养家心法

#### 当前实现
```python
financial_score = 60  # 默认基础分

if pd.notna(latest.get('roe')) and latest['roe'] > 10:
    financial_score += 10

if pd.notna(latest.get('or_yoy')) and latest['or_yoy'] > 10:
    financial_score += 10

if pd.notna(latest.get('profit_to_gr_yoy')) and latest['profit_to_gr_yoy'] > 10:
    financial_score += 10
```

#### 养家心法标准
根据SOUL.md第5条和第6条：

**财务质量评分标准：**
- ROE>15%、毛利率>30%、营收增长>10%、净利润增长>10%
- 低于标准的降分

#### 核心差异

| 维度 | 当前标准 | 养家心法标准 | 问题 |
|------|---------|-------------|------|
| 基础分 | 60分 | 0分 | 基础分过高，所有股票至少及格 |
| ROE标准 | >10%加分 | >15%才达标 | 标准过低 |
| 营收增长 | >10%加分 | >10%是及格线 | 没有分级 |
| 净利润增长 | >10%加分 | >10%是及格线 | 没有分级 |
| 毛利率 | 只采集不评分 | >30%才达标 | 完全忽略 |
| 资产负债率 | 只采集不评分 | 需要评分 | 完全忽略 |
| 扣分机制 | 无 | 负增长扣分 | 缺失 |

#### 实际影响
- **问题公司也能及格：** 即使ROE为5%、营收负增长的公司也能拿到60分
- **缺乏区分度：** 优质公司和问题公司得分差距小
- **违背养家心法：** 第3条"数据求真原则"，第5条"行业成长系数分级"

### ❌ 问题2：龙头属性评分缺失

#### 当前实现
```python
dragon_score = 60  # 潜在龙头（固定值）
```

#### 养家心法标准
根据SOUL.md第7条：

**龙头属性评分：**
- 板块龙一：100分
- 板块龙二：80分
- 市场龙头：90分
- 潜在龙头：60分
- 跟风股：0分（坚决回避）

**龙头识别标准：**
- 板块内第一个涨停
- 封板最坚决（开盘即涨停或早盘快速封板）
- 对板块带动效应最强

**纪律：** "有龙一，不看龙二；有龙二，不看跟风"

#### 核心差异
- 当前固定给60分（潜在龙头），完全没有识别逻辑
- 无法区分龙头和跟风股
- 无法落实"有龙一，不看龙二；有龙二，不看跟风"的纪律

### ❌ 问题3：情绪周期评分过于简化

#### 当前实现
```python
current_emotion, emotion_score = get_market_emotion()
```

根据之前的分析，情绪周期判断过于简单：
- 只基于上证指数涨跌幅
- 缺少涨停板数量、跌停板数量、核按钮等核心数据
- 硬编码阈值，不够科学

#### 养家心法标准
根据SOUL.md第8条：

**情绪周期评分：**
- 绝望：0分（冰点期，等待拐点）
- 退潮：20分（减仓或空仓）
- 高潮：40分（警惕风险）
- 发酵：60分（可参与但控制仓位）
- 启动：80分（黄金期，大胆出击）
- 冰点：100分（拐点临近，准备抄底）

#### 核心差异
- 缺少多维度数据（涨停热度、亏钱效应、资金活跃度、趋势指标）
- 无法准确判断市场处于哪个阶段
- 情绪评分不够科学

### ❌ 问题4：综合评分公式错误

#### 当前实现
```python
long_term_score = growth_score  # 只看行业成长系数
medium_term_score = (theme_heat_index * 0.5) + (financial_score * 0.3) + (tech_score * 0.2)
short_term_score = (tech_score * 0.8) + (emotion_score * 0.2) - market_cap_penalty
```

#### 养家心法标准
根据SOUL.md第10条：

**长期综合得分：**
- 长期综合得分 = 行业成长系数（0-100分）
- 高成长（1.0）= 100分
- 中成长（0.7）= 70分
- 低成长（0.3）= 30分
- 无成长（0.1）= 10分

**中期综合得分：**
- 中期综合得分 = 题材热度指数×50% + 财务质量分×30% + 中期趋势分×20%
- 题材热度指数 = (近期涨停热度分 × 30%) + (催化逻辑分 × 30%) + (行业成长系数 × 40%)

**短期综合得分：**
- 短期综合得分 = 技术面分×80% + 情绪周期分×20% - 市值扣分
- 技术面分 = 250天均线得分 × 50% + 缠论买点得分 × 50%

#### 核心差异

| 维度 | 当前 | 养家心法 | 问题 |
|------|-----|---------|------|
| 中期趋势分 | tech_score（技术面） | 中期趋势分（未定义） | 概念混淆 |
| 题材热度指数 | ✅ 正确 | ✅ 正确 | 无问题 |
| 情绪周期分 | emotion_score | emotion_score | 无问题 |
| 技术面分 | tech_score（外部计算） | 250天均线得分×50%+缠论买点得分×50% | 需要验证 |
| 市值扣分 | ✅ 正确 | ✅ 正确 | 无问题 |

**问题：** 中期综合得分中使用了`tech_score`（技术面），但养家心法要求使用"中期趋势分"。这两个概念不同：
- 技术面分：250天均线得分 + 缠论买点得分
- 中期趋势分：应该反映股票的中期走势，如30日均线、60日均线等

## 修改建议

### 优先级1：重构财务评分（高优先级）

#### 新评分标准（0-100分）

**1. 从0分开始，而非60分基础分**
```python
financial_score = 0  # 从0分开始
```

**2. 多维度综合评分**

**盈利能力（40分）：**
- ROE（15分）：>20%得15分，>15%得12分，>10%得8分，>5%得4分，≤5%得0分
- 毛利率（15分）：>50%得15分，>40%得12分，>30%得8分，>20%得4分，≤20%得0分
- 净利率（10分）：>15%得10分，>10%得8分，>5%得5分，≤5%得0分

**成长性（30分）：**
- 营收增长（15分）：>30%得15分，>20%得12分，>10%得8分，>0%得4分，≤0%得-5分
- 净利润增长（15分）：>30%得15分，>20%得12分，>10%得8分，>0%得4分，≤0%得-5分

**安全性（20分）：**
- 资产负债率（15分）：<40%得15分，<60%得10分，<80%得5分，≥80%得0分
- 流动比率（5分）：>2得5分，>1.5得3分，>1得1分，≤1得0分

**质量与可持续性（10分）：**
- 经营现金流/净利润（5分）：>1得5分，>0.8得3分，>0.5得1分，≤0.5得0分
- ROE连续性（5分）：连续3年>15%得5分，连续3年>10%得3分，否则得1分

**3. 有加分有扣分**
- 负增长扣分：营收或净利润负增长，各扣5分
- 财务恶化扣分：ROE连续下降，扣5分

**3. 识别龙头**
```python
def identify_dragon(stock_code, limit_up_stocks):
    """
    识别龙头属性
    """
    if len(limit_up_stocks) == 0:
        return 0, '跟风股'  # 板块无涨停，不是龙头

    if stock_code not in limit_up_stocks:
        return 0, '跟风股'  # 自己没涨停，不是龙头

    # 获取涨停股票的详细信息
    stock_details = []
    for code in limit_up_stocks:
        daily_data = pro.daily(ts_code=code, trade_date=today)
        if len(daily_data) > 0:
            stock_details.append({
                'code': code,
                'open': daily_data.iloc[0]['open'],
                'high': daily_data.iloc[0]['high'],
                'low': daily_data.iloc[0]['low'],
                'close': daily_data.iloc[0]['close'],
                'vol': daily_data.iloc[0]['vol']
            })

    # 判断是否是龙一
    dragon1 = None
    dragon2 = None

    # 按涨停时间排序（开盘时间越早越可能是龙一）
    stock_details.sort(key=lambda x: x['vol'])  # 临时用成交量排序

    if len(stock_details) >= 1:
        dragon1 = stock_details[0]['code']

    if len(stock_details) >= 2:
        dragon2 = stock_details[1]['code']

    # 判断龙头属性
    if stock_code == dragon1:
        return 100, '板块龙一'
    elif stock_code == dragon2:
        return 80, '板块龙二'
    else:
        return 0, '跟风股'
```

## 总结

### 核心问题

1. **财务评分逻辑严重不符合养家心法**
   - 基础分60分过高
   - 标准低于养家心法要求
   - 缺少扣分机制
   - 毛利率、负债率只采集不评分

2. **龙头属性评分缺失**
   - 固定给60分
   - 无法识别龙头和跟风股

3. **情绪周期评分过于简化**
   - 只基于指数涨跌幅
   - 缺少多维度数据

4. **中期综合得分公式概念混淆**
   - 使用技术面分代替中期趋势分

### 修改优先级

1. **高优先级：** 重构财务评分、实现龙头属性识别
2. **中优先级：** 完善情绪周期判断
3. **低优先级：** 修正中期综合得分公式

### 风险提示

- **财务数据可能造假**：需要结合技术面和情绪面综合判断
- **单一季度数据可能失真**：需要看趋势
- **不同行业需要不同标准**：不能一刀切

---

**结论：当前评分系统严重偏离养家心法标准，必须按照上述方案进行重构。**
