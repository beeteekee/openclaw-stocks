## 项目角色

### 项目经理（你）
**职责**：
- 任务分配和协调

**职责**：
- 前端页面开发（*.html, *.js, *.css）

- 数据可视化
- 用户交互优化
- **只负责前端代码，不触碰后端代码和项目管理文件**

## 工作流程

### 1. 任务分配流程

**项目经理 → 后端Agent**：
- 需求描述：实现某个功能或修复bug
- 技术要求：明确输入输出、性能要求
- 优先级：紧急/高/中/低
- 截止时间：预估完成时间

**项目经理 ← 后端Agent**：
- 进度报告：当前完成度
- 遇到问题：技术难点、依赖问题
- 解决方案：建议的修复方案
- 代码审查：自检报告

**项目经理 → 前端Agent**：
- 需求描述：UI/UX需求
- 设计稿：参考图或文字描述
- API依赖：需要哪些后端接口
- 优先级：紧急/高/中/低

**项目经理 ← 前端Agent**：
- 进度报告：当前完成度
- 设计稿：实现的效果图
- API需求：需要后端提供哪些接口
- 问题反馈：样式、交互问题

### 2. 协作流程

**后端-前端协作**：
1. 项目经理定义API接口（参数、返回格式）
2. 后端Agent实现API
3. 前端Agent调用API
4. 联调测试
5. 修复bug

**协作示例**：
```
项目经理：需要新增"批量查询"接口

后端Agent：
- 理解需求
- 设计API: POST /api/analyze/batch
- 实现代码
- 测试验证
- 报告完成

前端Agent：
- 理解需求
- 设计批量输入UI
- 调用API
- 展示结果
- 报告完成

项目经理：
- 验证功能
- 记录到记忆
```

## 技术决策记录

### 决策1：使用Flask作为后端框架
**原因**：
- 轻量级，易于部署
- 适合小型项目
- 生态丰富

### 决策2：使用原生JavaScript开发前端
**原因**：
- 保持轻量级
- 无需构建工具
- 易于维护

### 决策3：使用JSON文件存储统计数据
**原因**：
- 简单易用
- 无需数据库
- 适合小型应用

### 决策4：使用ECharts进行数据可视化
**原因**：
- 功能强大
- 中文文档完善
- 社区活跃

## 风险管理

### 技术风险
- **风险**：Tushare API限流
- **应对**：添加缓存，减少API调用

### 性能风险
- **风险**：大量数据计算导致响应慢
- **应对**：优化算法，使用异步处理

### 兼容性风险
- **风险**：不同浏览器兼容性问题
- **应对**：使用polyfill，测试主流浏览器
